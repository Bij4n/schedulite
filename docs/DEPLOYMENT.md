# Deployment Guide

This guide covers deploying Schedulite to a production environment. The application is designed to run on any platform that supports Ruby, PostgreSQL, and Redis.

## Prerequisites

- PostgreSQL 16+
- Redis 7+
- Ruby 3.2+
- Node.js 18+ (for asset compilation)
- A Twilio account with SMS-enabled phone number
- A Square developer account (for gift card feature)
- A domain name with SSL certificate

## Option A: Fly.io

### Initial Setup

```bash
# Install flyctl
curl -L https://fly.io/install.sh | sh

# Login and create app
fly auth login
fly launch --name schedulite --region iad --no-deploy
```

### Provision Services

```bash
# PostgreSQL
fly postgres create --name schedulite-db --region iad
fly postgres attach schedulite-db

# Redis
fly redis create --name schedulite-redis --region iad
```

### Configure Secrets

```bash
fly secrets set \
  LOCKBOX_MASTER_KEY="$(ruby -e "require 'securerandom'; puts SecureRandom.hex(32)")" \
  RAILS_MASTER_KEY="$(cat config/master.key)" \
  APP_HOST="https://schedulite.fly.dev" \
  REDIS_URL="redis://default:password@schedulite-redis.fly.dev:6379"
```

Set Twilio and Square credentials:

```bash
fly secrets set \
  TWILIO_ACCOUNT_SID="AC..." \
  TWILIO_AUTH_TOKEN="..." \
  TWILIO_PHONE_NUMBER="+15550001234" \
  SQUARE_ACCESS_TOKEN="..." \
  SQUARE_LOCATION_ID="..."
```

### fly.toml

```toml
[build]

[env]
  RAILS_ENV = "production"
  RAILS_LOG_TO_STDOUT = "true"
  RAILS_SERVE_STATIC_FILES = "true"

[http_service]
  internal_port = 3000
  force_https = true
  auto_stop_machines = true
  auto_start_machines = true

[[services]]
  internal_port = 3000
  protocol = "tcp"

  [[services.ports]]
    port = 80
    handlers = ["http"]

  [[services.ports]]
    port = 443
    handlers = ["tls", "http"]

[checks]
  [checks.health]
    port = 3000
    type = "http"
    interval = "15s"
    timeout = "2s"
    path = "/up"
```

### Deploy

```bash
fly deploy
fly ssh console -C "bin/rails db:migrate"
fly ssh console -C "bin/rails db:seed"  # optional, for demo data
```

### Sidekiq Worker

Create a separate process for Sidekiq in your `Procfile`:

```
web: bin/rails server -p 3000
worker: bundle exec sidekiq
```

Or run it as a separate Fly machine:

```bash
fly scale count web=1 worker=1
```

## Option B: Render

### Setup

1. Create a new **Web Service** from your GitHub repo
2. Set the build command: `bundle install && npm install && npm run build && npm run build:css && bin/rails assets:precompile`
3. Set the start command: `bin/rails server -p $PORT`

### Add Services

- **PostgreSQL**: Create a Render PostgreSQL database, copy the internal connection URL
- **Redis**: Create a Render Redis instance
- **Background Worker**: Create a new Worker service with start command `bundle exec sidekiq`

### Environment Variables

Set these in the Render dashboard:

```
DATABASE_URL=<from Render PostgreSQL>
REDIS_URL=<from Render Redis>
LOCKBOX_MASTER_KEY=<generate with ruby -e "require 'securerandom'; puts SecureRandom.hex(32)">
RAILS_MASTER_KEY=<contents of config/master.key>
APP_HOST=https://your-app.onrender.com
RAILS_ENV=production
```

Plus Twilio and Square credentials as listed above.

## Option C: VPS / Docker

### Dockerfile

The project includes a `Dockerfile` generated by Rails. Build and run:

```bash
docker build -t schedulite .
docker run -p 3000:3000 \
  -e DATABASE_URL="postgres://user:pass@host/schedulite_production" \
  -e REDIS_URL="redis://host:6379/0" \
  -e LOCKBOX_MASTER_KEY="..." \
  -e RAILS_MASTER_KEY="..." \
  -e RAILS_ENV=production \
  schedulite
```

### Docker Compose (development)

```yaml
version: "3.8"
services:
  web:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgres://schedulite:password@db/schedulite_development
      - REDIS_URL=redis://redis:6379/0
      - LOCKBOX_MASTER_KEY=your_key_here
    depends_on:
      - db
      - redis

  worker:
    build: .
    command: bundle exec sidekiq
    environment:
      - DATABASE_URL=postgres://schedulite:password@db/schedulite_development
      - REDIS_URL=redis://redis:6379/0
      - LOCKBOX_MASTER_KEY=your_key_here
    depends_on:
      - db
      - redis

  db:
    image: postgres:16
    environment:
      POSTGRES_USER: schedulite
      POSTGRES_PASSWORD: password
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine

volumes:
  pgdata:
```

## Production Checklist

### Security

- [ ] `LOCKBOX_MASTER_KEY` is set and backed up securely (losing it means losing access to encrypted PHI)
- [ ] `RAILS_MASTER_KEY` is set (for Rails credentials decryption)
- [ ] Force SSL is enabled (`config.force_ssl = true` in production.rb)
- [ ] HSTS headers are configured
- [ ] Secure cookies are enabled
- [ ] Session timeout is 15 minutes (configured via Devise `:timeoutable`)

### Database

- [ ] PostgreSQL 16+ with `pgcrypto` extension
- [ ] Connection pooling configured (PgBouncer recommended for >50 concurrent users)
- [ ] Automated backups enabled
- [ ] Point-in-time recovery configured

### Twilio

- [ ] Production Twilio account (not test credentials)
- [ ] SMS-enabled phone number provisioned
- [ ] Webhook URL configured: `https://your-domain.com/webhooks/twilio`
- [ ] If HIPAA required: Twilio BAA signed (available on paid plans)

### Square

- [ ] Production Square credentials (not sandbox)
- [ ] Gift card flow tested end-to-end
- [ ] Note: Square does not currently sign BAAs — gift card API calls must never include PHI (only token + amount)

### Monitoring

- [ ] Sentry DSN configured for error tracking
- [ ] Health check endpoint (`/up`) monitored
- [ ] Sidekiq dashboard accessible (mount at `/sidekiq` for admin users)
- [ ] Log aggregation configured (Lograge outputs structured JSON)

### DNS & SSL

- [ ] Domain pointed to deployment platform
- [ ] SSL certificate provisioned (auto via Let's Encrypt on Fly/Render)
- [ ] `APP_HOST` env var set to production domain

## Ongoing Operations

### Database Migrations

```bash
# Fly.io
fly ssh console -C "bin/rails db:migrate"

# Render
# Migrations run automatically on deploy via build command

# Docker
docker exec schedulite-web bin/rails db:migrate
```

### Sidekiq Monitoring

Mount the Sidekiq web UI in routes (admin-only):

```ruby
# config/routes.rb
require "sidekiq/web"
authenticate :user, ->(user) { user.owner? || user.admin? } do
  mount Sidekiq::Web => "/sidekiq"
end
```

### Backups

The `LOCKBOX_MASTER_KEY` is critical. If lost, all encrypted PHI fields become permanently unreadable. Store it in:
- A password manager (1Password, Vault, etc.)
- A sealed envelope in a physical safe
- Never in source control or unencrypted storage

### Data Retention

Completed appointments are retained for 7 years by default (configurable per tenant). The `DataRetentionJob` (scheduled via Sidekiq-Cron or similar) handles automated purging. Audit logs are never purged.
